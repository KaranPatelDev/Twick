Project Overview

Twick is a comprehensive social media platform built with Django that replicates and extends Twitter's functionality with modern features and robust architecture.

Modern Technology Stack

Built with Django 5.1.1, Python 3.12+, Bootstrap 5, and PostgreSQL for a robust and scalable foundation.

Social Features

Complete social media functionality including tweets, likes, comments, follows, direct messaging, and real-time notifications.

Enterprise Ready

Production-ready with Docker deployment, comprehensive testing, security features, and detailed documentation.

Core Features

Everything you need for a complete social media platform

Tweet Management

Create, edit, delete tweets with image support, character limits, and privacy controls.

Engagement System

Like, comment, reply system with real-time updates and notification system.

Follow System

Follow/unfollow users with support for private accounts and follow requests.

Direct Messaging

Private messaging system with group chats and file sharing capabilities.

Hashtag System

Automatic hashtag detection, trending topics, and hashtag-based discovery.

Notifications

Real-time notifications for likes, comments, follows, and direct messages.

Search & Discovery

Advanced search functionality for users, tweets, and hashtags with filters.

User Profiles

Comprehensive user profiles with avatars, bios, and customizable themes.

Quick Installation

Get Twick up and running in minutes with Docker or local development setup.

Prerequisites: Docker and Docker Compose installed on your system.
1
Clone the Repository

Download the Twick source code to your local machine

git clone https://github.com/yourusername/twick.git
2
Navigate to Directory

Change to the project directory

cd twick
3
Build and Start

Build Docker images and start all services

docker-compose up --build
Ready!

The application will be available at http://localhost:8000

Open Application
Prerequisites: Python 3.12+, pip, and virtualenv installed.
1
Clone Repository

Download the source code

git clone https://github.com/yourusername/twick.git && cd twick
2
Create Virtual Environment

Isolate project dependencies

python -m venv twick_env
3
Activate Environment

Activate the virtual environment

source twick_env/bin/activate
On Windows: twick_env\Scripts\activate
4
Install Dependencies

Install required Python packages

pip install -r requirements.txt
5
Setup Database

Initialize database and create admin user

python manage.py migrate
python manage.py createsuperuser
6
Run Development Server

Start the Django development server

python manage.py runserver
Development Ready!

Access your application at http://127.0.0.1:8000

Open Application
Production Setup: Includes PostgreSQL, Redis, and Nginx configuration.
1
Production Deployment

Deploy using production Docker Compose configuration

docker-compose -f docker-compose.prod.yml up --build -d
2
Alternative: Production Script

Use the automated production setup script

chmod +x start_prod.sh
./start_prod.sh
3
Environment Configuration

Set up production environment variables

cp .env.example .env
Edit .env with your production settings (database URL, secret key, etc.)
4
SSL Certificate (Recommended)

Set up HTTPS with Let's Encrypt

certbot --nginx -d yourdomain.com
Replace yourdomain.com with your actual domain
Production Ready!

Your Twick application is now running in production mode with all optimizations enabled.

✓ Docker Containers ✓ Database ✓ Redis Cache ✓ Nginx Proxy

API Documentation

RESTful API endpoints for all Twick functionality.

Authentication

Twick uses Django's built-in authentication system with session-based authentication for web interface and token-based authentication for API access.

Method Endpoint Description Authentication
POST /register/ User registration Not required
POST /login/ User login Not required
POST /logout/ User logout Required

Tweet Management

Create, read, update, and delete tweets with full CRUD operations.

Method Endpoint Description Authentication
GET /tweets/ List all tweets Optional
POST /tweets/create/ Create new tweet Required
GET /tweet/<id>/ Get tweet details Optional
PUT /tweet/<id>/edit/ Edit tweet Required (Owner)
DELETE /tweet/<id>/delete/ Delete tweet Required (Owner)
POST /tweet/<id>/like/ Like/unlike tweet Required

User Management

User profile management and social connections.

Method Endpoint Description Authentication
GET /profile/<username>/ View user profile Optional
POST /profile/edit/ Edit own profile Required
POST /profile/<username>/follow/ Follow/unfollow user Required
GET /profile/<username>/followers/ List user followers Optional

Production Ready

Production-optimized Docker images with security best practices and performance tuning.

Production Setup
docker-compose.yml - Complete containerized setup
version: '3.8'
services:
web:
build:
context: .
dockerfile: Dockerfile
ports:
- "8000:8000"
environment:
- DEBUG=True
- DATABASE_URL=postgresql://postgres:postgres@db:5432/twick
- REDIS_URL=redis://redis:6379/0
depends_on:
- db
- redis
volumes:
- ./media:/app/media
- ./static:/app/static

db:
image: postgres:15
environment:
- POSTGRES_DB=twick
- POSTGRES_USER=postgres
- POSTGRES_PASSWORD=postgres
volumes:
- postgres_data:/var/lib/postgresql/data
ports:
- "5432:5432"

redis:
image: redis:7-alpine
ports:
- "6379:6379"

volumes:
postgres_data:
Container Features
  • Multi-stage builds for optimized production images
  • Health checks for service monitoring and reliability
  • Volume management for persistent data and media files
  • Environment variables for configuration management
  • Network isolation for security and service communication
