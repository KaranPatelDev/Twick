Getting Started

Quick setup guide to get Twick running on your local machine in minutes

Prerequisites

What you'll need

Make sure you have these installed before getting started:

Development Tools

  • Python 3.8+ - Programming language
  • Git - Version control
  • Code Editor - VS Code, PyCharm, etc.
  • Terminal/Command Line - For running commands

Optional (Recommended)

  • Docker & Docker Compose - Containerization
  • PostgreSQL - Production database
  • Redis - Caching and sessions
  • Node.js - Frontend build tools

Quick Start (5 minutes)

The fastest way to get Twick running using Docker:

1

Clone the Repository

Terminal
git clone https://github.com/yourusername/twick.git
cd twick
2

Start with Docker

Terminal
docker-compose up -d

This will start all services including the database, Redis, and the Django application.

3

Access the Application

Open your browser and go to:

http://localhost:8000
That's it!

You should now see the Twick homepage. You can register a new account or use the admin interface at /admin.

Manual Installation

For development or if you prefer not to use Docker:

1

Clone & Navigate

Terminal
git clone https://github.com/yourusername/twick.git
cd twick
2

Create Virtual Environment

Windows
python -m venv venv
venv\Scripts\activate
macOS/Linux
python3 -m venv venv
source venv/bin/activate
3

Install Dependencies

Terminal
pip install -r requirements.txt
4

Setup Database

Terminal
python manage.py migrate
python manage.py createsuperuser
python manage.py collectstatic
5

Run Development Server

Terminal
python manage.py runserver

Visit http://127.0.0.1:8000

Environment Configuration

For proper setup, you may need to configure environment variables:

Create .env file:
# Database
DATABASE_URL=sqlite:///db.sqlite3
# DATABASE_URL=postgresql://user:password@localhost:5432/twick

# Security
SECRET_KEY=your-secret-key-here
DEBUG=True

# Email (for user verification)
EMAIL_BACKEND=django.core.mail.backends.console.EmailBackend
# EMAIL_HOST=smtp.gmail.com
# EMAIL_PORT=587
# EMAIL_HOST_USER=your-email@gmail.com
# EMAIL_HOST_PASSWORD=your-app-password

# Media Storage
MEDIA_URL=/media/
STATIC_URL=/static/

# Redis (for caching and sessions)
REDIS_URL=redis://localhost:6379/0

# Social Auth (optional)
# GOOGLE_OAUTH2_KEY=your-google-client-id
# GOOGLE_OAUTH2_SECRET=your-google-client-secret

First Steps After Installation

Create Admin User

Create a superuser account to access the admin interface:

Terminal
python manage.py createsuperuser
Load Sample Data

Load some sample data for testing:

Terminal
python manage.py loaddata fixtures/sample_data.json
Admin Interface

Access the Django admin at:

http://localhost:8000/admin
API Documentation

Explore the API endpoints at:

http://localhost:8000/api/docs

Common Issues

If port 8000 is already in use, try:

Use Different Port
python manage.py runserver 8080
Windows - Find & Kill Process
netstat -ano | findstr :8000
taskkill /PID [PID_NUMBER] /F
macOS/Linux - Find & Kill Process
lsof -i :8000
kill -9 [PID_NUMBER]

If you encounter migration issues:

Reset Migrations (Development Only)
python manage.py migrate --fake tweet zero
python manage.py migrate
Delete & Recreate Database (Development Only)
rm db.sqlite3
python manage.py migrate
python manage.py createsuperuser

If CSS/JS files aren't loading:

Collect Static Files
python manage.py collectstatic --clear
Note: Make sure DEBUG=True in settings for development
Check STATIC_URL and STATICFILES_DIRS in settings.py
