Installation Issues

Problem:

Getting errors related to Python version or package compatibility.

Solution:
Check Python Version
python --version # Should be 3.8 or higher
Create Virtual Environment with Specific Python
python3.9 -m venv venv
source venv/bin/activate # Linux/macOS
venv\Scripts\activate # Windows
Install Python 3.8+:
• Windows: Download from python.org
• macOS: brew install python@3.9
• Ubuntu: sudo apt install python3.9

Problem:

Package installation fails with permission errors or dependency conflicts.

Solutions:
Upgrade pip first
python -m pip install --upgrade pip
Install with user flag (permission issues)
pip install --user -r requirements.txt
Clear pip cache
pip cache purge
Install packages individually
pip install django
pip install django-extensions
# ... continue with other packages

Problem:

Virtual environment not working or packages not found.

Solutions:
Check activation: You should see (venv) in your terminal prompt
Activate on Windows
venv\Scripts\activate
Activate on macOS/Linux
source venv/bin/activate
Recreate Virtual Environment (Linux/macOS)
deactivate
rm -rf venv
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
Recreate Virtual Environment (Windows)
deactivate
rmdir /s venv
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt

Database Issues

Problem:

Database migration fails or shows conflicting migrations.

Solutions:
# Check migration status
python manage.py showmigrations

# Reset specific app migrations (DEVELOPMENT ONLY)
python manage.py migrate tweet zero
python manage.py migrate

# Reset all migrations (DEVELOPMENT ONLY)
rm db.sqlite3
find . -path "*/migrations/*.py" -not -name "__init__.py" -delete
find . -path "*/migrations/*.pyc" -delete
python manage.py makemigrations
python manage.py migrate

# For production, create a new migration file
python manage.py makemigrations --merge

Problem:

SQLite database is locked or in use by another process.

Solutions:
# Stop all Django processes
# Kill any runserver processes
ps aux | grep manage.py  # Find the process ID
kill -9 [PID]

# Windows: Use Task Manager or
tasklist | findstr python
taskkill /PID [PID] /F

# If database is still locked, restart your computer
# Or delete the database file (development only)
rm db.sqlite3
python manage.py migrate

Problem:

Getting "table doesn't exist" errors when accessing the application.

Solutions:
# Run migrations
python manage.py migrate

# If that doesn't work, check if migrations exist
python manage.py showmigrations

# Create migrations if they don't exist
python manage.py makemigrations

# Then migrate
python manage.py migrate

# Create superuser after migrations
python manage.py createsuperuser

Server Issues

Problem:

Cannot start server because port 8000 is already in use.

Solutions:
# Use a different port
python manage.py runserver 8080
python manage.py runserver 0.0.0.0:8000

# Find and kill process using port 8000
# Windows
netstat -ano | findstr :8000
taskkill /PID [PID_NUMBER] /F

# macOS/Linux
lsof -i :8000
sudo kill -9 [PID_NUMBER]

# Or use a specific port range
python manage.py runserver 8001

Problem:

Django development server fails to start with various errors.

Solutions:
# Check for syntax errors in settings
python manage.py check

# Validate models
python manage.py check

# Run with verbose output to see detailed errors
python manage.py runserver --verbosity=2

Problem:

Stylesheets and JavaScript files are not loading, resulting in broken layout.

Solutions:
# Collect static files
python manage.py collectstatic --clear

# Check settings.py for correct static file configuration
STATIC_URL = '/static/'
STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]

# For development, make sure DEBUG=True
DEBUG = True

# Clear browser cache or open in incognito mode
# Check browser developer tools for 404 errors

Problem:

User uploaded images and media files are not displaying.

Solutions:
# Check media settings in settings.py
MEDIA_URL = '/media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')

# Make sure URLs are configured in main urls.py
from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    # ... your url patterns
]

if settings.DEBUG:
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)

Docker Issues

Problem:

Docker containers fail to start or build properly.

Solutions:
# Stop all containers and rebuild
docker-compose down
docker-compose build --no-cache
docker-compose up -d

# Check logs for specific service
docker-compose logs web
docker-compose logs db

# Remove all containers and volumes (WARNING: destroys data)
docker-compose down -v
docker system prune -af

Problem:

Django app cannot connect to PostgreSQL database in Docker.

Solutions:
# Wait for database to be ready
# Add health checks to docker-compose.yml
depends_on:
  db:
    condition: service_healthy

# Use database hostname as defined in docker-compose.yml
# In settings.py:
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'twick',
        'USER': 'postgres',
        'PASSWORD': 'postgres',
        'HOST': 'db',  # This should match service name
        'PORT': '5432',
    }
}
